# portal-m-pages

## 项目简介

`portal-m-pages` 是一个用于归档和发布多个独立 HTML 页面的仓库。  
本仓库不针对任何特定域名或具体网站，仅作为静态页面集合的存储与展示空间。

## 目录结构

```
portal-m-pages/
├── index.html           # 仓库入口页（可选）
├── pages/               # 存放独立 HTML 页面
│   ├── example-01.html
│   ├── example-02.html
│   └── ...
├── assets/              # 公共资源（样式、脚本、图片等）
│   ├── css/
│   ├── js/
│   └── images/
└── README.md            # 本文件
```

- `pages/`：核心目录，每个 HTML 文件代表一个独立页面。
- `assets/`：存放页面共享的静态资源，便于统一管理。
- `index.html`：可选入口文件，用于导航或汇总。

## 页面归档说明

- 归档的页面均为独立静态文件，不依赖后端服务。
- 每个页面应保持自包含性，外部资源（如 CSS、JS）建议统一存放在 `assets/` 中。
- 页面文件名建议使用英文小写，以连字符分隔，保持清晰可读。
- 如需添加新页面，直接在 `pages/` 下创建新的 HTML 文件即可。

## 维护说明

- 欢迎提交 Pull Request 添加或更新页面，但请确保内容不违反平台政策。
- 请勿在页面中加入任何形式的追踪、广告或恶意代码。
- 资源文件请放入对应子目录，避免根目录杂乱。
- 本仓库不承诺内容长期可用性或更新频率。

## 使用方式

可通过 GitHub Pages 或其他静态托管服务直接部署，将仓库根目录作为站点根目录即可。  
例如，访问 `https://<username>.github.io/portal-m-pages/pages/example-01.html` 可查看对应页面。
